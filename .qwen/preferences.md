## Обязательное согласование перед выполнением

⚠️ **Правило**: Прежде чем выполнять задачу, я должен:

1. **Спросить пользователя** — уточнить детали и подтвердить готовность к выполнению
2. **Проверить папку `stitch/`** — найти соответствующий шаблон для задачи
3. **Применить шаблон** — использовать стили, названия и паттерны из шаблонаcl
✅ **Включено**: Автоматическое создание плана задач

При получении новой задачи:
1. Анализировать сложность
2. Создать план в папке `plans/` с именем `YYYY-MM-DD_task-name.md`
3. Показать план пользователю на подтверждение
4. Ждать подтверждения перед началом выполнения
5. После подтверждения — выполнять по плану

## Формат файла плана

```markdown
# План: <название задачи>

## Задачи
- [ ] Шаг 1
- [ ] Шаг 2
- [ ] Шаг 3

## Статус: Ожидает подтверждения / В работе / Завершён
```

## Папка для планов

`plans/` в корне проекта (`d:\Ispoved\plans\`)
