# Автоматическая выгрузка версий

## Обзор

Скрипт `release.js` автоматически обновляет версию приложения, собирает проект и выгружает изменения в GitHub.

---

## Быстрый старт

### 1. Настройте `.env` файл

Создайте файл `.env` в корне проекта:

```env
GITHUB_API_TOKEN=ghp_ваш_токен
GITHUB_REPO_OWNER=ваш-username
GITHUB_REPO_NAME=ispoved
```

**Как получить токен:**
1. GitHub → Settings → Developer settings → Personal access tokens
2. Generate new token (classic)
3. Выберите_scope `repo` (Full control of private repositories)
4. Скопируйте токен и вставьте в `.env`

## Использование

### Обновление PATCH версии (2.0.13 → 2.0.14)

```bash
npm run release
```

### Обновление MINOR версии (2.0.13 → 2.1.0)

```bash
npm run release:minor
```

### Обновление MAJOR версии (2.0.13 → 3.0.0)

```bash
npm run release:major
```

---

## Что делает скрипт

1. ✅ **Обновляет версию** в файлах:
   - `package.json` → `version`
   - `src/version.js` → `APP_VERSION`
   - `android/app/build.gradle` → `versionName` и `versionCode`

2. ✅ **Собирает проект**:
   - `npm run build` → Vite build + Capacitor sync

3. ✅ **Обновляет CHANGELOG.md**:
   - Добавляет новую запись с датой и версией

4. ✅ **Коммитит и пушит** в GitHub:
   - `git add .`
   - `git commit -m "release: версия X.X.X"`
   - `git push origin main`

## Ручное обновление CHANGELOG

Если нужно добавить описание изменений вручную:

1. Откройте `CHANGELOG.md`
2. Найдите запись с вашей версией
3. Добавьте секции:

```markdown
## [2.0.14] - 26 марта 2026

### Добавлено
- Новая функция X
- Улучшение Y

### Изменено
- Исправление Z
- Обновление стиля A

### Исправлено
- Баг #123
- Проблема с рамками
```

## Решение проблем

### ❌ Ошибка: GITHUB_API_TOKEN не найден

**Решение:** Создайте файл `.env` с токеном (см. выше)

### ❌ Ошибка: Permission denied (publickey)

**Решение:** Используйте HTTPS вместо SSH:
```bash
git remote set-url origin https://github.com/username/ispoved.git
```

### ❌ Ошибка сборки

**Решение:** Запустите сборку вручную для проверки:
```bash
npm run build
```

## Структура версионирования

| Тип | Команда | Пример | Когда использовать |
|-----|---------|--------|-------------------|
| **PATCH** | `npm run release` | 2.0.13 → 2.0.14 | Исправления багов, мелкие правки |
| **MINOR** | `npm run release:minor` | 2.0.13 → 2.1.0 | Новые функции, обратная совместимость |
| **MAJOR** | `npm run release:major` | 2.0.13 → 3.0.0 | Критические изменения, ломает совместимость |

## Дополнительные команды

```bash

# Синхронизация с Capacitor (без обновления версии)
npm run cap:sync

# Запуск dev-сервера
npm run dev
```

## Советы

1. ✅ **Проверяйте изменения перед выгрузкой:**
   ```bash
   git status
   git diff
   ```

2. ✅ **Тестируйте сборку локально:**
   ```bash
   npm run build
   npm run preview
   ```

3. ✅ **Создавайте теги для версий:**
   ```bash
   git tag v2.0.14
   git push origin v2.0.14
   ```

4. ✅ **Регулярно делайте бэкапы:**
   ```bash
   git branch backup-2026-03-26
   ```

## Поддержка

При возникновении проблем:
1. Проверьте `.env` файл
2. Убедитесь, что токен действителен
3. Проверьте права доступа к репозиторию
4. Запустите с флагом `--verbose` для отладки
