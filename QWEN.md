# Исповедь (Ispoved) — Проект контекста

## Обзор проекта

**Исповедь** — это православное мобильное веб-приложение (PWA) и гибридное приложение (Capacitor) для подготовки к таинству исповеди и изучения церковного календаря. Приложение помогает верующим подготовиться к исповеди, предоставляет ежедневные чтения, молитвы и информацию о святых.

### Основные возможности

- **Церковный календарь** — ежедневные чтения, память святых, информация о постах и седмицах
- **Подготовка к исповеди** — интерактивный список грехов с категориями, возможность создания личного списка
- **Молитвенное правило** — каноны (покаянный, Богородице, Ангелу Хранителю), последование ко Причащению
- **Телепромптер** — автопрокрутка текста молитв с регулируемой скоростью
- **Аудио-сопровождение** — воспроизведение молитв и чтений (TTS и аудиофайлы)
- **Экспорт в PDF** — генерация печатной версии списка исповеди
- **Мультиязычность** — поддержка русского и английского языков
- **Тёмная/светлая тема** — дизайн в стиле "The Sacred Digital Archive"

## Сборка и запуск

### Разработка
```bash
npm run dev              # Запуск Vite dev-сервера (порт 5173)
```

### Сборка
```bash
npm run build            # Синхронизация версии + Vite build + Capacitor sync
```

### Мобильная разработка
```bash
npm run cap:sync         # Синхронизация с Capacitor (android)
npm run sync-version     # Только синхронизация версии
npm run sync-github      # Коммит и пуш в GitHub (требует .env с токеном)
```

### Предварительный просмотр
```bash
npm run preview          # Preview production сборки
```

## Конфигурация

### Переменные окружения
Файл `.env` (не коммитить в репозиторий):
```bash
GITHUB_API_TOKEN=ghp_...
GITHUB_REPO_OWNER=your-username
GITHUB_REPO_NAME=ispoved
```

### Версионирование
Версия синхронизируется автоматически между:
- `package.json` → `version`
- `src/version.js` → `APP_VERSION`
- `android/app/build.gradle` → `versionCode` / `versionName`

Формат `versionCode`: `major * 10000 + minor * 100 + patch`
- Пример: `2.0.13` → `20013`

## Архитектурные особенности

### Глобальные функции
- `stopAllAudio()` — универсальная остановка всего аудио (TTS, Audio, плееры)
- `toggleSpeech(text, iconId, audioIndex)` — управление синтезом речи и аудиофайлами
- Профиль пользователя хранится в `localStorage` (`profile`: 'adult' | 'child' | 'monastic')

### Хранение данных
- **localStorage** — настройки, выбранные грехи, профиль, тема
- **JSON/JS модули** — статические данные (грехи, молитвы, переводы)
- **Индексированные данные** — грехи с категориями и объяснениями

### Capacitor плагины
- `@capacitor/status-bar` — управление статус-баром
- `@capgo/capacitor-navigation-bar` — навигационная панель Android
- `@capacitor-community/text-to-speech` — синтез речи
- `@aparajita/capacitor-biometric-auth` — биометрическая аутентификация
- `@capacitor/share` — шеринг контента
- `@capacitor/geolocation`, `@capacitor/motion` — датчики устройства

## Практики разработки

### Стиль кода
- **ES Modules** — только `import`/`export`, тип `module` в package.json
- **Vanilla JS** — без фреймворков, прямой DOM-манипуляции
- **Tailwind** — утилитарные классы для стилизации
- **CSS-переменные** — централизованное управление темой

### Тестирование
- Формальных тестов нет
- Проверка через `npm run build` (отсутствие ошибок компиляции)
- Ручное тестирование в браузере и на Android-устройстве

### Вклад в проект
1. Внести изменения в исходный код
2. Запустить `npm run sync-github` для автоматического обновления версии и CHANGELOG
3. Изменения автоматически коммитятся и пушатся в GitHub

## Особые заметки

### Аудио-система
- Все аудио останавливается через `stopAllAudio()` перед запуском нового
- Fallback на локальные файлы при недоступности серверов Azbyka.ru
- Интеллектуальное переключение между TTS и аудиофайлами

### Телепромптер
- Перетаскиваемая панель (touch/mouse)
- Авто-скрытие через 4 секунды бездействия
- Скорость: 0–500 px/sec (база 250 px/sec при 50%)
- Позиция сохраняется в `localStorage`

### Профили пользователей
- **adult** — полный список грехов (по умолчанию)
- **child** — сокращённый список для детей
- **monastic** — расширенный список для монашествующих

### PWA возможности
- Offline-работа через Service Worker (workbox)
- Кэширование шрифтов Google Fonts (1 год)
- Установка на устройство как нативное приложение
- Автообновление при новой версии

## Автоматическое планирование задач

✅ **Включено**: При получении новой задачи автоматически создавать план и сохранять в `plans/`

### Процесс:
1. Анализ задачи → определение сложности
2. Создание файла `plans/YYYY-MM-DD_task-name.md`
3. Показ плана пользователю → ожидание подтверждения
4. После подтверждения → выполнение по плану

### Формат плана:
```markdown
# План: <название задачи>

## Задачи
- [ ] Шаг 1
- [ ] Шаг 2
- [ ] Шаг 3

## Статус: Ожидает подтверждения / В работе / Завершён
```
